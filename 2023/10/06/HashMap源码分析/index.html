HashMap 简介

本篇使用的jdk源码版本jdk11

HashMap 是 Java 中的一个重要数据结构，它用于存储键值对（key-value pairs），并提供了高效的查找、插入和删除操作。HashMap 是常用的Java集合之一，是非线程安全的。

HashMap 继承了 Map、 Cloneable、Serializable 等接口

  • Map ：该接口提供了最基本键值对映射的基本框架，定义了键和值之间的映射关系，并提供了一系列的操作来管理这些映射关系
  • Cloneable : 也是一个标记接口，表明它具有拷贝能力，可以进行深拷贝和浅拷贝
  • Serializable : 也是一个标记接口，表明它可以进行序列化操作，也就是可以将对象转化为字节流进行持久化存储或者在网络中传输，非常方便
  • AbstractMap : 一个抽象类，它提供了一些通用的方法和骨架实现

注意 ：HashMap 是可以添加null值的，作为键值null值只能添加一个，但作为值可以添加多个

在看到这里，你可以会有以下几个疑问，让我一一为你解答

  1. 为什么 HashMap 的大小必须是2的幂

    • 在 HashMap 中，我们对元素的存储位置需要进行取模运算来获得，而在取模运算中hash % size 对于2的幂来说，可以等价于使用位运算 hash & (size - 1)，这可以保证哈希码的高位和低位都参与了桶索引的计算，从而是元素分布更均匀，减少了Hash碰撞的发生。
    • 使用2的幂作为大小可以提高散列函数的计算效率，因为计算 hash & (size - 1) 比计算 hash % size 更快。

  2. 什么是转载因子(LOAD_FACTOR)

    • 在HashMap中，转载因子通常用来衡量散列表的填充程度。即键值对数量相对于 HashMap 的容量比例。在Java的HashMap实现中，默认的转载因子是0.75，这意味着当HashMap中的键值对数量达到容量的75%时，HashMap会自动进行扩容操作。合理的装载因子可以减少空间还足够充足的时候发生扩容从而导致的空间浪费，也可以减少元素过多而空间不足时发生的Hash冲突发生的频率

  3. 什么是 Bin ，即桶

    • Bin 或桶是HashMap内部用来存储键值对的数据结构，用于解决哈希冲突。在Java 8之前，它是链表，而在Java 8及以后，它可以是链表或红黑树，具体取决于桶内键值对的数量。

  4. 为什么要将存储结构转换为红黑树

    • 当多个键映射到同一个桶时，如果链表过长，查找、插入和删除操作可能会变得很慢，因为它们需要遍历整个链表。红黑树是一种自平衡二叉搜索树，它在平均情况下具有较快的查找、插入和删除操作，因此可以在某些情况下提高HashMap的性能。

HashMap 的核心方法

RESIZE 方法

在初始化或者容量不够的时候，HashMap 都会调用这个方法来进行扩容，这个过程会遍历 hash 表中的所有元素并对其进行 hash 分配，是非常耗时的。在编写程序中，要尽量避免 resize。resize 方法实际上是将 table 初始化和 table 扩容 进行了整合，底层的行为都是给 table 赋值一个新的数组。

PUT 方法

  • afterNodeAccess 是java.util.LinkedHashMap类中的一个方法，用于在访问（查找、读取）映射中的节点（键值对）后执行一些操作。LinkedHashMap是HashMap的一个子类，它扩展了HashMap的功能，提供了按照插入顺序或访问顺序（LRU，最近最少使用）来迭代元素的能力。
  • afterNodeInsertion 与 afterNodeAccess 同理，在数据插入后进行回调
