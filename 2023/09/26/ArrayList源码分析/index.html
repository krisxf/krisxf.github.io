ArrayList 简介

本篇使用的jdk源码版本jdk11

ArrayList是一个非常常用的类，它提供了一种便捷的方式来管理和操作动态数组。它的底层实现是数组队列，相当于动态数组，比起常用的数组，ArrayList可以动态的增长容量。

ArrayList继承了AbstractList类，并实现了 List、RandomAccess、Cloneable、Serializable 等接口

  • AbstractList : 一个抽象类，它提供了一些通用的方法和骨架实现
  • List : 表明它是一个列表，支持添加，删除，查询等操作
  • RandomAccess : 这是一个标记接口（Marker Interface），它在Java标准库中用于标识实现了随机访问的数据结构。在ArrayList 中，我们即可以通过元素的下标快速的获取元素对象，这就是快速随机访问。在 LinkedList 中，由于LinkedList 底层是由链表实现的，不支持快速的获取元素对象，所以它并没有实现 RandomAccess 这个接口
  • Cloneable : 也是一个标记接口，表明它具有拷贝能力，可以进行深拷贝和浅拷贝
  • Serializable : 也是一个标记接口，表明它可以进行序列化操作，也就是可以将对象转化为字节流进行持久化存储或者在网络中传输，非常方便

注意 ：** ArrayList 是可以添加null值得，但是不建议往其中添加null值，会使代码更加不容易维护**

关于ArrayList的一些问题

为什么ArrayList 要实现 AbstractList ，不能直接实现 List 接口就行了嘛？

  • 代码复用和一致性：
    • AbstractList 提供了 List 接口的骨架实现，这意味着所有实现 AbstractList 的子类都可以共享一些通用的列表操作逻辑，例如 size() 和 isEmpty() 等方法。这样可以避免在每个具体的列表实现中重复编写相同的代码，提高了代码复用性。
    • 这也有助于确保所有的 List 实现都具有一致的行为，因为它们都共享了来自 AbstractList 的相似的实现。
  • 简化实现：
    • List 接口定义了许多方法，其中一些可能需要复杂的数据结构和算法来实现。使用 AbstractList 可以将列表的实现细节分解为更小的部分，简化了实现的过程，使其更易于理解和维护。
    • 子类只需要关注实现 AbstractList 抽象类所要求的方法，而不需要关心其他不相关的方法。
  • 提供默认实现：
    • AbstractList 中提供了默认的实现，这使得实现 List 接口的类可以选择性地覆盖这些默认实现。如果某个具体的列表类不需要对某些方法进行特定的优化或自定义实现，可以直接继承 AbstractList 的默认实现，减少了编码工作。
  • 接口的演化：
    • Java 编程语言的版本在不断演化，新的版本可能会添加新的方法到接口中。使用 AbstractList 可以减轻在现有 List 实现中添加新方法所带来的负担，因为子类只需要实现新添加的方法，而不需要修改已经存在的方法。

elementData 是 ArrayList 中用来存储数据的数组，既然 ArrayList 实现了 Serializable 接口，又为什么要将存储序列化的数组使用 transient 来修饰呢(即不会将这个数组参与序列化)

  • 容量和性能优化：
    • elementData 数组的容量可能大于实际存储的元素数量，因为 ArrayList 通常会分配比实际元素数量更大的容量，以减少频繁的扩容操作，提高性能。
    • 如果 elementData 数组参与序列化，那么在反序列化时，可能会导致反序列化后的 ArrayList 对象具有不必要的大容量。这可能会浪费内存，尤其是在存储大型 ArrayList 对象时。
  • 序列化和反序列化的一致性：
    • 序列化的目的是将对象的状态保存到持久存储，然后在需要时恢复它。然而，ArrayList 的容量状态通常不是用户关心的对象状态的一部分。用户更关心的是元素的内容。
    • 如果 elementData 数组参与序列化，那么序列化和反序列化操作可能会包括大量不必要的数据，这不仅浪费了时间和资源，而且可能导致不必要的数据传输。
  • 数组可能包含敏感数据：
    • 在某些情况下，elementData 数组可能包含敏感数据，例如密码、密钥或其他敏感信息。如果将这些数据参与序列化，可能会引发安全问题，因为序列化的数据可能在传输或存储过程中暴露。

在 ArrayList 的有参构造函数中，为什么要专门判断一下初始化参数是否为0，不能直接 new 一个 初始容量为 0 长度的数组吗

  • 如果初始化参数为0，这意味着用户希望创建一个空的 ArrayList，没有初始容量。在这种情况下，为了节省内存和不浪费不必要的空间，ArrayList 不会立即分配一个初始容量为0的数组。相反，它会使用一个共享的空数组（EMPTY_ELEMENTDATA），这个数组在初始状态下不占用额外的内存空间，因为它是一个静态常量。

ArrayList 的最大数组大小为什么不是 Integer.MAX_VALUE 而是 Integer.MAX_VALUE - 8

  • 头部开销：在Java中，数组通常有一些头部开销，用于存储数组的长度和其他元数据。这些开销通常是一个整数的大小，所以减去 8 可以确保即使数组达到 Integer.MAX_VALUE 的上限，仍然可以保留足够的空间来存储元数据。
  • 内存对齐：在某些系统上，数组的大小可能需要按照某种特定的内存对齐规则进行分配。减去 8 可以确保数组大小在内存对齐规则下仍然有效。
  • 避免潜在问题：在某些情况下，如果数组的大小接近 Integer.MAX_VALUE，可能会导致性能问题或内存溢出。通过将 MAX_ARRAY_SIZE 设置为 Integer.MAX_VALUE - 8，可以在靠近数组大小限制的情况下提前触发警告或错误，以防止出现不可预测的问题。
